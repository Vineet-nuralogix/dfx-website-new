WMS FAQ

What is Web Measurement Service?​

Web Measurement Service offers the same features related to measuring health indicators as our other offerings and is designed for easy integration into external partner’s web application. It delivers a web “measurement gateway” on the 3rd party URL (sub domain).

How is Web Measurement Service different from your other offerings?​

It is specifically designed for easy integration into external partner’s web application. It delivers a web “measurement gateway” on the 3rd party URL (sub domain).

What kind of profile information do you need to carry out the measurements?​

WMS uses user-provided information to further enhance the accuracy of measurements. Basic user demographic information is used to provide more accurate blood pressure and body shape measurements:

Age

Sex at Birth (Male or Female)

Height

Weight

Additionally, user may be prompted for a 3-question medical questionnaire to provide a more accurate cardiovascular disease assessment:

Do you smoke? (Yes / No)

Are you taking blood pressure medication? (Yes / No)

Are you diabetic? (No / Type 1 / Type 2)

What is CNAME?​

A Canonical Name record (abbreviated as CNAME record) is a type of resource record in the Domain Name System (DNS) that maps one domain name (an alias) to another (the canonical name).

By using CNAME we are able to offer seamless transition to the WMS. The URL in the browser remains the same for the user as that of third-party website.

Do you redirect the customers to your own website for measurement?​

On submitting the profile information, the user is redirected to WMS hosted at http://nuralogix.ai via a third-party subdomain (using CNAME) but the URL in the browser remains the same for the user as that of third-party website.

Which browsers do you currently support?​

Officially supported browsers are:

Google Chrome on Windows, Android and

Apple Safari on macOS and iOS

Other browsers based on Chromium 83 and above, should also work but are not officially supported.

Which languages can WMS user interface support?​

The WMS currently supports English, Chinese, Spanish, Portuguese, Japanese

Is it necessary to collect user profile information through a form if we already have that information stored with us?​

Third party website can provide this data as per their preference. They may ask the user to enter the information through a profile entry form, or they may already be having this data stored with them, or they may adopt any other method of obtaining this information.

If a user has multiple cameras in his device, then is it possible to select any one of them?​

Before starting measurement, the user is presented with a list available cameras on his device and can select any one of them.

Do you offer a sample application to help us get started?​

Please contact NuraLogix to get access to the WMS sample application which demonstrates the call-in / callback functionality of Web Measurement Service (WMS).

Please visit sample application section in our WMS docs for more details.

Is the user profile information encrypted while being transferred to you?​

Profile information will be passed from the app to WMS through a specially crafted URL (the "call-in" URL). Since this URL will contain personal information, it will be encrypted so that only a specially configured WMS will be able to read the information.

What is a Config ID?​

To get going, you need a WMS CONFIG ID - a unique identifier that is linked to your app's specific configuration.

Please check out the getting started section of WMS documentation for more details.

Can I use WebView inside my app to integrate the Web Measurement Service?​

It is possible to do so, but WebView is not a recommended way to use WMS and you are encouraged to check out other better ways offered by us to integrate our service into your Android and iOS apps.

How can I integrate WMS into my website or app?​

Please check out the getting started section of WMS documentation.

How are the measurement results returned to our website or app?​

Measurement results are sent back to the user through a callback URL. User is redirected back to the third-party website once DeepAffex™ has completed processing the measurement results.

Partner web application is responsible for all admin/display/storage of end-user measurement results. They may use the results as per their choice of implementation.

How can I customize the look of the WMS according to my brand?​

In order to customize the style of the measurement screen to match your brand identity, a stylesheet can be provided to control the colors and background images.

Please checkout styling section of WMS documentation.
