DFX Cloud API FAQs

What is DFX Cloud API?​

The DFX Cloud API enables customers to communicate with our DFX cloud service. It allows them to programmatically access and manage their organization’s DeepAffex™ account data and results.

The DFX Cloud API is published as a reference specification (documentation) describing the programmatic interfaces (or endpoints) that are provided by the DeepAffex™ cloud engine.

These network endpoints allow a remote client (program) to invoke and access all DeepAffex™ micro-services and features.

The DFX Cloud API allows the use of REST for synchronous (blocking) transactions as well as Web-socket protocols for asynchronous (non-blocking) communications.

What is an Organization on DFX?​

In the DeepAffex™ Cloud's business-to-business services model, the Organization is the partitioning "silo" that gets created and licensed by a business.

All measurements, results, raw data etc. that are created under an Organization are secure and isolated from other Organizations. Only NuraLogix™ can create Organizations.

What is a DFX license key and why do I need it?​

The use of the DFX network API is restricted to customers holding a valid DeepAffex™ license. The DFX license key permits a specified quantity (as licensed) of client devices to communicate with the DFX API for a specified duration (as licensed) while accessing a specified ORGANIZATION (as licensed).

What is meant by tokens while using DFX service?​

Authentication on the DeepAffex™ Cloud uses JSON Web Tokens.

Please check out the Tokens section in our developer guider.

What is a device token?​

A device token usually represents a single device. A device token provides access to endpoints that allow for device-specific features e.g. making measurements.

Please check out the Tokens section in our developer guider for more information.

What is a user token?​

A user token provides access to most non-measurement related endpoints. To obtain a user token, login as a user using the Users.Login endpoint.

Please check out the Tokens section in our developer guider for more information.

Is user token mandatory?​

The response to Organizations.RegisterLicense contains a RoleID for the license - if it is SDK_DEVICE, then obtaining a user token is mandatory.

Please check out the More details and best practices section in our docs.

Please explain the License API registration flow?​

The DFX license key permits a specified quantity (as licensed) of client devices to communicate with the DFX API for a specified duration (as licensed) while accessing a specified ORGANIZATION (as licensed).

Every physical device (handset, desktop, laptop) that requires client communication with DFX must first register with the API to receive a unique device ‘token’.

The client software submits the DFX license key to the API to request registration of a device.

The API responds by issuing a valid device ‘token’ to the client software. The client software should save this device ‘token’ (persistent storage or cache) and submit the device ‘token’ with subsequent API transactions.

Each device registered causes the available device count balance to decrement (-1/device) from the original licensed count, until all devices have been assigned.

At any time devices that are no longer required may be ‘unregistered’ with the API in order to reclaim and increment (+1/device) the remaining device balance count.

What is One-Time Refresh Token (OTRT)?​

One-Time Refresh Token (OTRT) has been introduced to improve token security.

How to FAQs

How can I start using the DFX Cloud API?​

Please check out this Getting started with the cloud API page.

How can I obtain a device token?​

To obtain a device token for the device in use, register your organizations License Key on the DeepAffex™ Cloud using the Organizations.RegisterLicense endpoint.

Please check out the Tokens section in our developer guider for more information.

How do I register a device to carry out the measurements?​

To register a device you need to obtain a device token. A device token usually represents a single device. It provides access to endpoints that allow for device-specific features e.g. making measurements. To obtain a device token for the device in use, register your organization license on the DeepAffex™ Cloud.

Please check out the Register endpoint on DeepAffex™ Cloud API Reference for more details.

How many devices can I register for measurement through my license?​

The number of devices are mapped to your license.

You can check the maximum number of devices as well as the count of registered devices in your DFX dashboard. Click on the licenses option in the left navigation bar. The counts can be seen under the “DEVICE REGISTRATIONS“ heading.

How can I check the number of devices registered through my license?​

You can check the maximum number of devices as well as the count of registered devices in your DFX dashboard. Click on the licenses option in the left navigation bar. The counts can be seen under the “DEVICE REGISTRATIONS“ heading.

How can I unregister a device?​

Some License Keys have a limited number of allowed active Devices. If you need to use your License on a different device, make sure to unregister your current device first by calling the Organizations.UnregisterLicense endpoint.

How can I re-register a device?​

To ensure that a token is valid, you can call the Verify Token endpoint. If it is not valid then you may register the device again following the normal registration process.

Please check out the Register endpoint on DeepAffex™ Cloud API Reference for more details.

How can I know about the latest updates to the DFX Cloud API?​

Please check out our release notes to know about the latest updates.

Can a device token get expired? How can I ensure that a token is still valid?​

Device tokens may expire. To ensure that a token is valid, you can call the Verify Token endpoint endpoint. If the ActiveLicense field in the response is false or if the API returns one of the following errors: INVALID_TOKEN, LICENSE_EXPIRED, INACTIVE_LICENSE or INACTIVE_DEVICE, then the client must re-register the license.

Please check out the Register License endpoint on DeepAffex™ Cloud API Reference for more details.

Please checkout the User Logout section in our docs.

How can I decode real-time results?​

Please check out the Decoding real-time results section in our developer guide.

How can I send data to the cloud after subscribing to the results?​

Please check out the Adding data section in our developer guide.

Troubleshooting FAQs

Why my API requests to register the DFX license from a device keep failing?​

One possibility is that you have exhausted your count (quantity) of licensed devices.

Every physical device (handset, desktop, laptop) that requires client communication with DFX must first register with the API to receive a unique device ‘token’. The maximum number of devices that are permitted to connect with DFX is controlled by your license-key.

You can make requests to the API to ‘unregister’ any/all currently registered device ‘tokens’ and reclaim device balance.

Why I am getting “INCORRECT_REQUEST“ for a WebSocket request?​

Requests that do not abide by the prescribed Request structure are not processed and a response with the following body is returned:

{

Code: "INCORRECT_REQUEST"

}
